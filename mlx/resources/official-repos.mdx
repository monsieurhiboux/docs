Quand vous cherchez une information fiable sur MLX, commencez par les dépôts officiels de l'organisation [`ml-explore`](https://github.com/ml-explore).

Ils vous donnent la vision la plus stable de l'écosystème :

- code source
- exemples officiels
- documentation
- discussions

## Le repo à ouvrir selon votre besoin

| Besoin | Repo à ouvrir d'abord |
|--------|------------------------|
| Comprendre MLX lui-même | `mlx` |
| Générer du texte ou fine-tuner un LLM | `mlx-lm` |
| Voir des implémentations complètes | `mlx-examples` |
| Intégrer MLX dans une app Swift | `mlx-swift` et `mlx-swift-examples` |

---

## Pourquoi les exemples officiels comptent autant

Les tutoriels communautaires sont utiles, mais ils vieillissent vite. Les exemples officiels vous aident à vérifier :

- les API réellement actuelles
- les patterns de chargement recommandés
- la structure attendue par les outils

Quand vous hésitez entre deux approches, alignez-vous d'abord sur le repo officiel le plus proche de votre cas.

---

## Suivre les mises à jour sans bruit

Si vous utilisez MLX régulièrement :

- surveillez les releases de `mlx`
- jetez un oeil aux issues et discussions de `mlx-lm`
- consultez les exemples Swift officiels avant d'implémenter votre propre architecture

Ce réflexe réduit le risque de copier une approche devenue obsolète.

---

## Point d'attention

L'écosystème bouge vite. Quand une page communautaire donne une commande différente de celle du repo officiel, privilégiez le repo officiel.
