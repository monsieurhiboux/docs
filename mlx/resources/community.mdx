## Reddit

### r/LocalLLaMA — La Communauté Centrale

[reddit.com/r/LocalLLaMA](https://reddit.com/r/LocalLLaMA) — **266 000+ membres**

C'est **le** hub pour tout ce qui touche aux LLMs locaux. La communauté partage :
- Des configs hardware et benchmarks
- Des recommandations de modèles selon le hardware
- Des tutoriels de fine-tuning
- Des comparatifs de frameworks
- Des cas d'usage créatifs

**Thread incontournable** : ["Lessons learned so far: LoRA Fine tuning on MacBook Air with MLX"](https://reddit.com/r/LocalLLaMA/comments/18wabkc)

Leçons clés du thread :
1. Valider toujours les fichiers JSONL avant l'entraînement
2. Fermer les apps gourmandes en RAM pendant l'entraînement
3. Limiter les exemples à ~3500 tokens pour éviter les erreurs mémoire
4. Vitesse attendue : 60-80 tokens/sec sur M1 MacBook Air 16GB
5. MLX utilise vraiment le GPU Metal, contrairement à certaines configs llama.cpp

## Newsletters & Blogs

### Simon Willison — La Pépite ⭐

[simonwillison.net](https://simonwillison.net) / [Substack](https://simonw.substack.com)

Simon Willison est l'une des voix les plus suivies dans la communauté dev/IA. Il écrit régulièrement sur MLX et les LLMs locaux avec une approche très pratique.

**Article phare** : ["Run LLMs on macOS using llm-mlx and Apple's MLX framework"](https://simonwillison.net/2025/Feb/15/llm-mlx/) — publié le 15 février 2025

Extraits de ses retours :
- **Mistral-Small-24B** : "semble vraiment de qualité GPT-4 avec seulement ~12 GB de RAM — un excellent modèle par défaut"
- **DeepSeek-R1-Distill-Qwen-32B-4bit** : "excellent pour le raisonnement, produit des blocs `<think>...</think>` fascinants"
- **Llama 70B** : "nécessite 64 GB de RAM mais très capable — utiliser `llm chat` pour garder le modèle chargé"

Simon maintient aussi le plugin [llm-mlx](https://github.com/simonw/llm-mlx) — une des façons les plus simples de gérer des modèles MLX.

## Recherche Académique

**[Production-Grade Local LLM Inference on Apple Silicon](https://arxiv.org/abs/2511.05502)** (arxiv, 2025)

Étude comparative rigoureuse de MLX, MLC-LLM, Ollama, llama.cpp et PyTorch MPS sur Apple Silicon. Résultats clés :
- MLX a le throughput le plus élevé
- MLC-LLM offre le TTFT le plus court sur certains contextes
- llama.cpp est le plus efficace pour les use-cases légers
- Ollama est optimal pour l'ergonomie développeur
