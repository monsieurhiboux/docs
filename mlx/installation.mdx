## Avant de commencer

Ce guide cible le cas le plus simple et le plus courant :

- un Mac avec Apple Silicon
- un Python natif ARM
- un environnement virtuel dédié au projet

<Warning>
Si votre terminal ou votre Python tourne sous Rosetta, vous risquez des erreurs de dépendances ou des performances incohérentes. Vérifiez d'abord votre environnement.
</Warning>

### Vérifier l'architecture de Python

```bash
python -c "import platform; print(platform.machine())"
```

Sur Apple Silicon, vous voulez voir `arm64`.

Si ce n'est pas le cas, installez un Python natif via Homebrew, `pyenv` ou le site officiel de Python, puis recréez votre environnement virtuel.

---

## Installer MLX dans un environnement propre

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
pip install mlx
```

### Vérifier l'installation

```bash
python -c "import mlx.core as mx; print(mx.array([1, 2, 3]))"
```

Si vous voyez un tableau s'afficher, MLX est installé correctement.

## Installer `mlx-lm`

`mlx-lm` ajoute les outils orientés LLM :

- génération de texte
- chargement de modèles depuis Hugging Face
- conversion et quantification
- fine-tuning LoRA et QLoRA
- serveur local compatible API

```bash
pip install -U mlx-lm
```

Si vous prévoyez de télécharger, publier ou manipuler des datasets, ajoutez aussi :

```bash
pip install pandas huggingface_hub "huggingface_hub[cli]"
```

---

## Premier test avec un modèle local

```bash
mlx_lm.generate \
  --model mlx-community/Llama-3.2-3B-Instruct-4bit \
  --prompt "Explique MLX en deux phrases simples."
```

Au premier lancement, `mlx-lm` télécharge le modèle depuis Hugging Face puis le réutilise depuis le cache local.

<Tip>
Pour découvrir MLX, commencez avec un modèle compact en `4bit`. Le téléchargement est plus léger et vous évitez les erreurs mémoire sur une machine modeste.
</Tip>

## Connexion à Hugging Face

Vous pouvez travailler sans compte pour beaucoup de modèles publics. En revanche, connectez-vous si vous voulez :

- accéder à un dépôt privé
- publier un modèle converti
- partager un modèle fine-tuné

```bash
huggingface-cli login
```

## Installer les outils Swift

Si votre objectif principal est une app Swift qui charge un modèle MLX, ajoutez le package `mlx-swift` dans Xcode ou via Swift Package Manager :

```swift
dependencies: [
    .package(
        url: "https://github.com/ml-explore/mlx-swift",
        branch: "main"
    )
]
```

<Info>
Si vous voulez utiliser le modèle système Apple dans une app Swift, vous n'avez pas besoin d'installer MLX. Regardez plutôt la section [Foundation Models](/mlx/foundation-models/introduction).
</Info>

## Problèmes fréquents

<AccordionGroup>
  <Accordion title="`pip` ne trouve pas le package" icon="triangle-exclamation">
    Vérifiez d'abord la version de Python et son architecture. Le problème vient souvent d'un Python non compatible ou lancé sous Rosetta.
  </Accordion>

  <Accordion title="`ModuleNotFoundError: No module named mlx`" icon="triangle-exclamation">
    Activez le bon environnement virtuel avant d'exécuter vos scripts :

    ```bash
    source .venv/bin/activate
    pip show mlx
    ```
  </Accordion>

  <Accordion title="Le chargement d'un modèle échoue par manque de mémoire" icon="triangle-exclamation">
    Commencez avec un modèle plus petit ou déjà quantifié en `4bit`. Fermez aussi les applications qui consomment beaucoup de mémoire unifiée.
  </Accordion>

  <Accordion title="Je développe en Swift et je ne sais pas par où commencer" icon="circle-question">
    Si vous voulez un modèle Apple prêt à l'emploi dans une app Swift, partez sur Foundation Models. Si vous voulez embarquer un modèle open weight précis, regardez `mlx-swift` ou une intégration HTTP vers un serveur local.
  </Accordion>
</AccordionGroup>
